The Team

We are a team of three 42 students who collaborated on building this HTTP/1.1 server implementation from scratch in C++. This project is part of the system programming track in the 42 curriculum.

About the Webserv Project

Webserv is a challenging project at 42 that requires students to create a fully-functional HTTP/1.1 server. The goal is to implement a server comparable to industry standards like Nginx or Apache, while gaining deep understanding of networking protocols and system programming.

Key features implemented in our Webserv:

  • HTTP/1.1 protocol compliance - Properly implemented request parsing and response generation
  • Multiple virtual servers - Support for running different server configurations on the same instance
  • Non-blocking I/O - Efficient handling of concurrent connections
  • Configurable routing - Flexible URL routing system
  • HTTP redirections - Full support for standard HTTP redirects
  • File uploads - Processing multipart form data
  • Custom error pages - Configurable error responses
  • CGI execution - Support for dynamic content generation

This project has been an excellent opportunity to learn about network programming, protocol implementation, and system design through practical implementation.

Technical Implementation

Our server is built with modern C++ and employs a non-blocking I/O model with an event-driven architecture. Key technical aspects include:

  • Event-driven architecture using select/poll/epoll
  • Connection state management
  • Efficient memory usage and buffer management
  • HTTP/1.1 protocol parsing and generation
  • MIME type detection and content handling
  • CGI process management
